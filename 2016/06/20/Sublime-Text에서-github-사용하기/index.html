  • git:push로 깃허브에 파일을 올리는 작업이 완료되면 콘솔창에 이런식으로 뜹니다.

미리 읽기

깃(Git)과 깃허브(GitHub)는 형상관리를 할수 있는 툴과 깃을 저장할 수 있는 공간입니다.
깃과 깃허브의 사용법에 대해서 알아보겠습니다.
먼저 깃(Git)의 흐름입니다.
Git으로 하는 흐름은 그림과 같이,
1. 워킹 디렉토리에서 파일을 수정2. staging area에 git add 명령어로 파일을 stage해서 커밋할 스냅샷을 만듬3. staging area에 있는 파일을 git commit명령어로 커밋해서 Git 디렉토리에 영구적인 스냅샷 저장의 순서로 이루어 집니다.
이 흐름은 로컬 데이터베이스에서 일어나는 흐름이라
