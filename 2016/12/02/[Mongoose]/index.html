  • 몽구스 핵심 이벤트 목록 - MG_EV_ACCEPT : 수신연결로 새 서버 연결을 수락할 때 발생한다. void ev_data는 피어의 공용 소켓 주소이다. - MG_EV_CONNECT : mg_connect()에 의해 생선된 새로운 아웃 바운드 연결이 실패하거나 성공했을 때 발생한다.void ev_data 는 int success 이다. success가 0 이면 연결이고, 그렇지 않으면 오류코드가 나온다. - MG_EV_RECV : 새로운 데이터가 수신되어 recv_mbuf의 끝에 추가된다.void ev_data 는 int received_bytes 이다. 일반적으로 이벤트 처리기는 nc-> recv_mbuf에서 수신 된 데이터를 확인하고 mbuf_remove ()를 호출하여 처리 된 데이터를 삭제하고 필요한 경우 연결 플래그 nc-> flags를 설정하고 (struct mg_connection 참조) mg_send ()와 같은 출력 함수를 사용하여 원격 피어에 데이터를 기록해야한다. - MG_EV_SEND : 몽구스는 원격 피어에 데이터를 쓰고 mg_connection :: send_mbuf에서 쓰여진 데이터를 버린다.void ev_data는 int num_sent_bytes 이다.※ 몽구스 출력함수는 mg_connection::send_mbuf에만 데이터를 추가한다. 실제 IO는 mg_mgr_poll()에 의해 수행된다. MG_EV_SEND 이벤트는 IO가 완료되었다는 알림이다. - MG_EV_POLL : mg_mgr_poll() 을 호출 할 때마다 모든 연결로 이벤트를 보낸다. 이 이벤트는 타임아웃 여부확인, close, 하트비트 등동작을 수행되는데 사용될 수 있다. - MG_EV_TIMER : mg_set_timer() 가 호출되면 연결로 보냅니다. # 플래그 - MG_F_FINISHED_SENDING_DATA는 모든 데이터가 send_mbuf에 추가되었음을 몽구스에게 알려줍니다. 몽구스가 그것을 소켓으로 보내 자마자 연결이 닫힙니다. - MG_F_BUFFER_BUT_DONT_SEND는 몽구스에게 send_mbuf에 데이터를 추가하도록 지시하지만 데이터를 나중에 수정하고 MG_F_BUFFER_BUT_DONT_SEND 플래그를 지워서 전송하기 때문에 send_mbuf에 데이터를 추가하지만 보류합니다. - MG_F_CLOSE_IMMEDIATELY는 Mongoose에게 보통 오류가 발생한 직후에 연결을 닫으라고 지시합니다.
